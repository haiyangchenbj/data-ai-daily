## 月度趋势分析

### 趋势一：数据平台 AI 能力从实验走向产品级交付

三大平台在 1 月的更新密度都很高，且共同特征是：**之前在预览的 AI 功能开始批量 GA。** Databricks One（统一发现入口）和 Lakebase（操作型数据库）同时 GA，Snowflake 的多项 AI 函数（AI_FILTER、AI_AGGR、AI_COUNT_TOKENS）GA，Fabric 的 Warehouse 结果集缓存和 MERGE GA。平台层的 AI 能力不再是路线图上的承诺，而是可以在生产环境使用的功能。

**So what：** 数据团队可以开始将 AI 功能纳入正式的技术选型评估，而不是仅在 sandbox 中试用。

### 趋势二：MCP 协议成为 Agent-数据平台连接的事实标准

1 月同时出现了三个 MCP 相关动作：Databricks 发布 MCP Servers 公开预览（Genie、向量搜索、DBSQL、Unity Catalog），Fabric Data Agent 支持作为 MCP 服务器在 VS Code 中使用，Snowflake 的语义视图增强也在为 Agent 消费数据做准备。MCP 正在从 Anthropic 推动的协议变成数据平台的接入标准。

**So what：** 如果你在评估 Agent 与数据平台的集成方案，MCP 是目前最值得押注的标准协议。

### 趋势三：传统数据库"AI 内置化"加速

Oracle AI Database 26ai 本地部署 GA 是 1 月最值得注意的单点事件。它把向量搜索、RAFT 复制、SQL 防火墙等能力直接内置到数据库内核，而不是作为外挂服务。这延续了 Oracle 一贯的"收敛"路线——所有能力都应在数据库内完成。

**So what：** 对于重度依赖 Oracle 的企业（金融、电信、政府），AI 能力的获取路径可能不需要引入新平台，而是通过现有数据库版本升级。

### 趋势四：云厂商涨价潮的序幕

AWS 1 月 23 日上调 EC2 机器学习容量块价格约 15%，Google Cloud 1 月 27 日宣布 5 月起数据传输涨价（北美翻倍）。这是 3 月全球涨价潮的序幕。Agent 驱动的推理算力需求暴涨是底层原因。

**So what：** FinOps 在 1 月还是"建议实践"，到 3 月已经变成刚需。
